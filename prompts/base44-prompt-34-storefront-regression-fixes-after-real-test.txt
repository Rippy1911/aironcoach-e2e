GOAL
═══════════════════════════════════════════════════════════════

Prompt 33 was directionally correct, but the implementation left admin blocks
and did not fully enforce mobile/storefront ordering.

Make a surgical follow-up fix:

  1. Remove owner admin blocks from ProfileCoachTab main content.
  2. Move owner controls into a compact top-right hero toolbar.
  3. Make right sidebar a pure coaching offer / conversion rail.
  4. On mobile, show the primary offer immediately under the hero.
  5. Keep profile facts (Experience, Location/mode) in main content, not as
     separate cards competing with offers on the right.

No drag & drop. No page builder.

FIX 2 — HERO OWNER TOOLBAR MUST BE TOP-RIGHT, NOT CONTENT FLOW
═══════════════════════════════════════════════════════════════

In pages/UserProfile:

Owner actions should be an absolutely/visually positioned toolbar in the hero:

  Desktop:
    top-right corner of hero card
    small buttons:
      Guest View

      optional more menu

Visitor CTAs remain in content flow:
  Follow
  Send message request
  Share

Owner toolbar acceptance:
  - owner sees Guest View / Copy profile link / Coach Hub
  - visitor does not
  - toolbar is anchored top-right on desktop hero
  - no duplicate "Open Coach Hub" appears below tabs/content

═══════════════════════════════════════════════════════════════
FIX 3 — RIGHT SIDEBAR = OFFER / CONVERSION RAIL ONLY
═══════════════════════════════════════════════════════════════

Current right column mixes:
  - Location & mode
  - price
  - Experience
  - Availability

Change it to a clear offer rail:

Right sidebar should contain:
  1. Featured coaching service card if services exist
  2. Additional service cards, if any
  3. Fallback price/contact card if no services exist
  4. CTA: Book a Session / Request coaching / Send message request
  5. Small availability summary only if directly relevant to booking

Right sidebar should NOT contain as separate cards:
  ✗ Experience
  ✗ Location & mode
  ✗ Certifications
  ✗ profile facts that belong in content

Move those to main content:
  Main content sections:
    About
    Specialties
    Experience
    Certifications
    Testimonials
    Location & mode

For the no-services fallback:
  Card title should be explicit:
    "Coaching offer" or "Coaching available"
  Show:
    price_per_session + currency
    "per session"
    CTA
    optional "Online" / accepting clients badge

The visitor should instantly understand:
  - what can I buy?
  - how much does it cost?
  - what button do I click?

═══════════════════════════════════════════════════════════════
FIX 4 — MOBILE ORDER: OFFER IMMEDIATELY UNDER HERO
═══════════════════════════════════════════════════════════════

Current visitor mobile order is wrong:

  Hero
  Tabs
  About
  Specialties
  Price card
  Experience

Required mobile order:

  Hero
  Primary offer card / fallback price card
  Tabs
  About
  Specialties
  Experience
  Certifications
  Testimonials
  Location & mode

Implementation guidance:

  Use CSS order classes or render a dedicated mobile offer block:

    <div className="lg:hidden">
      <CoachStorefrontSidebar variant="mobile-primary" />
    </div>

    <div className="hidden lg:block">
      <CoachStorefrontSidebar />
    </div>

  Avoid duplicating booking side effects. If duplicated visually, both CTAs
  call the same handler.

Acceptance:
  On 390px mobile viewport, the price/offer card appears before About and
  Specialties in scroll order.

FIX 5 — PROFILE STUDIO IS FOR ADMIN CONTROLS
═══════════════════════════════════════════════════════════════

Coach Hub → Profile Studio should be the place for:

  - public/private
  - accepting clients
  - headline / bio / profile identity
  - avatar / cover
  - location / mode
  - certifications / experience / testimonials

Services must remain separate:
  Coach Hub → Services or /CoachServices
