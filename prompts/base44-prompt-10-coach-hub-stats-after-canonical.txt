## Problem

After coach profile creation, `/CoachBusinessHub` has historically shown:

```txt
Create a coach profile first
```

or:

```txt
You're not a coach yet
```

even when coach fields exist.

Known root risks:

1. `getCoachStats` function can be missing/dead-deployed.
2. Frontend maps API failure to a ‚Äúnot a coach‚Äù empty state.
3. Coach profile save redirects before `myProfile` and `coachStats` cache are fresh.

---

## Required behavior

### Gate model

Use explicit states:

| State | Condition | UI |
|-------|-----------|----|
| No coach profile | canonical profile has no coach identity/capability | Create coach profile CTA |
| Coach profile saved, stats loading | coach profile exists + stats request pending | Loading dashboard |
| Coach profile saved, stats OK | stats API returns data | Dashboard |
| Coach profile saved, stats API error | API 404/500/network | Error state + Retry |
| Inactive/deactivated coach | `coach_status=inactive` or equivalent | Reactivate / settings CTA |

Never show ‚Äúnot a coach‚Äù for API failure.
