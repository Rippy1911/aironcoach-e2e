# P0 — ANTI-DUPLIKACJA CTA (ZRÓB PIERWSZE)

## Problem (screenshot użytkownika po publish)

Feed pokazuje jednocześnie:
- „Create post" w headerze (prawy górny róg)
- Inline composer „Share a tip, announcement or update…"
- „Create a post" w Quick Actions (prawy rail)

**To jest błąd UX. Napraw w tej turze.**

## Reguła (wszystkie 4 taby)

```
Jedna akcja = jeden primary entry point na ekranie.
Header CTA tylko gdy main column NIE MA już tej samej akcji.
Quick Actions = skróty do INNYCH tabów/routów, nie duplikat bieżącego taba.
```

## Instrukcje per tab

### Feed
- USUŃ header CTA „Create post" — CommunityHeaderCTA zwraca null dla tab=feed
- USUŃ „Create a post" z Quick Actions na Feedzie
- Inline composer = JEDYNY primary create post
- ComposeFAB: ukryj na desktop gdy composer widoczny; mobile OK jako skrót scroll-to-composer
- Header: tytuł „Community" + taby ONLY. BEZ subtitle. BEZ CTA.

### Teams
- Header „Create team" = OK
- Quick Actions: bez „Create team" jeśli jest w headerze
- Rail: Invite member, Find coach, Go to Feed (cross-tab)

### Messages
- Header „New message" = OK
- Quick Actions: bez „New message"
- Rail: Go to Feed, Go to teams, Find coach

### People
- Header „Invite" = OK
- Quick Actions: bez „Invite"
- Rail: Discover coaches, Go to Feed, My follows

## Pliki
- components/community/CommunityHeaderCTA.jsx
- components/community/CommunityShell.jsx
- components/community/FeedTab.jsx
- Quick Actions / discovery rail component

## Acceptance P0
- [ ] Feed: dokładnie 1 widoczny „create post" = composer
- [ ] Screenshot Feed: brak header CTA, brak Quick Actions create, jest composer
- [ ] Screenshot Teams/Messages/People: header CTA bez duplikatu w rail
- [ ] Raport: tabela tab → primary → co usunięte

# P1 — PROFILE HUB (duży blok — zrób kompletnie, nie half-build)

## Problem
Brak centrum tożsamości. Profil rozproszony: Settings, /UserProfile, /MyFollows, menu.

## B1. Menu — dopracuj linki

- „Follows & followers" → `/UserProfile?id=<self>&tab=follows` (NIE /MyFollows)
- Header popover klik → `/UserProfile?id=<self>`

## B2. UserProfile?id=<self> = Profile Hub

Dodaj taby (jeden komponent tab bar, wspólny layout):

| Tab | Zawartość |
|-----|-----------|
| **overview** | Bio, streak summary (własny exact OK), quick stats, Edit profile, link Settings |
| **activity** | Publiczne posty użytkownika (empty state jeśli brak) |
| **follows** | **Przenieś całą logikę z /MyFollows** — sub-taby Following (N) \| Followers (N), PersonCard, streak_tier_public |
| **stats** | Streak, workouts, volume — tylko własne dane |
| **coach** | Tylko gdy coach capability: bio, services, link Coach Hub |

Route: `/UserProfile?id=<profileId>&tab=<overview|activity|follows|stats|coach>`

**Follows tab szczegóły:**
- NIE duplikuj hero (avatar+imię już w profilu)
- PersonCard jak w Community
- Empty: „You're not following anyone yet" + CTA → /Community?tab=people
- Reuse getMyFollows (już naprawione bulkUpdate)

## B3. /MyFollows — backward compat

- Redirect do `/UserProfile?id=<self>&tab=follows`
- Nie usuwaj route od razu — redirect wystarczy

## B4. Profil innego usera

/UserProfile?id=<other>:
- Hero: avatar, imię, tier, streak_tier_public (opt-in), verified, Follow button
- Taby: overview \| activity (publiczne)
- NIE pokazuj prywatnych stats (exact streak, volume) bez opt-in

## Acceptance P1
- [ ] Własny profil: 4-5 tabów działa
- [ ] Follows tab = dawny MyFollows bez orphan pustej strony
- [ ] Menu → profil?tab=follows
- [ ] /MyFollows redirect
- [ ] Screenshot każdego taba profilu
