PROMPT 33 — Coach profile UX cleanup: storefront po lewej/prawej + edycja w Coach Hub

═══════════════════════════════════════════════════════════════
KONTEKST
═══════════════════════════════════════════════════════════════

Prompt 31/32 naprawiły owner edit na /UserProfile.
Teraz problem nie jest techniczny, tylko UX / product architecture:

  ✓ Owner widzi edycję
  ✗ Publiczny profil wygląda za bardzo jak panel administracyjny
  ✗ Elementy typu "Open Coach Hub" są w treści profilu
  ✗ Oferty / coaching services nie są domyślnie mocno widoczne po prawej
  ✗ Zniknął wcześniejszy panel marketing, który był dobry do edycji profilu coacha
  ✗ Wszystko inline na publicznym profilu jest zbyt ciężkie

NOWY KIERUNEK:
  Publiczny profil = clean storefront dla odwiedzającego
  Coach Hub = miejsce konfiguracji / edycji / operacji
  Services = osobna konfiguracja ofert, nie część marketing/personal info
  Bez drag&drop — zachowujemy simplicity, ale modern.

═══════════════════════════════════════════════════════════════
CELE UX
═══════════════════════════════════════════════════════════════

1. Profil coacha ma wyglądać jak nowoczesna strona twórcy / coacha:
   Patreon / Contra / Linktree / modern consultant profile.

2. Oferty, które coach tworzy, mają być widoczne od razu po prawej stronie:
   - cena
   - co zawiera
   - CTA: Book / Request coaching / Send request
   - status: accepting clients

3. Treść profilu ma być czysta:
   - About
   - Specialties
   - Certifications
   - Testimonials
   - Location / mode
   - Experience

4. Narzędzia właściciela nie powinny mieszać się z treścią:
   - "Open Coach Hub"
   - visibility toggles
   - publish/private controls
   - internal navigation

   To ma być w panelu właściciela, nie w środku contentu.

═══════════════════════════════════════════════════════════════
RIGHT SIDEBAR — COACHING SERVICES / OFFERS
═══════════════════════════════════════════════════════════════

Oferty tworzone w CoachServices powinny renderować się po prawej stronie profilu.

Jeśli coach ma services:
  - pokaż listę kart ofert
  - pierwsza / featured oferta na górze
  - każda karta:
      title
      short description
      price + currency
      cadence, jeśli istnieje (per session / monthly / package)
      key inclusions, max 3 bullets
      CTA

Jeśli coach nie ma services:
  - pokaż prosty fallback:
      "Coaching available"
      price_per_session jeśli istnieje
      CTA: "Send coaching request"
  - owner widzi small hint:
      "Add services in Coach Hub → Services"

VISITOR:
  - CTA: Book / Request coaching / Send message request
  - żadnych pencil icons
  - żadnych internal controls

OWNER:
  - sidebar nadal pokazuje oferty tak jak visitor
  - owner controls jako subtelne:
      "Manage services" link/button
      NIE inline form
  - link prowadzi do /CoachServices lub Coach Hub → Services

═══════════════════════════════════════════════════════════════
OWNER ACTIONS — PRZENIEŚĆ Z TREŚCI DO HERO / TOP RIGHT
═══════════════════════════════════════════════════════════════

Usuń z głównej treści profilu:
  ✗ Open Coach Hub card/button jako content block
  ✗ Profile & visibility card w main content
  ✗ Share my coach profile publicly toggle w treści
  ✗ Accepting new clients toggle w treści

Zamiast tego:

W prawym górnym rogu hero / bannera owner widzi mały owner toolbar:
  - Guest View toggle
  - Edit profile / Manage profile
  - Open Coach Hub
  - More menu (...)

Toolbar ma być wizualnie oddzielony od contentu:
  - small buttons
  - aligned top-right
  - not a big content card

Visitor nie widzi owner toolbar.

═══════════════════════════════════════════════════════════════
INLINE EDIT — ZMNIEJSZYĆ, NIE USUWAĆ
═══════════════════════════════════════════════════════════════

Inline edit zostaje, ale nie ma być jedynym sposobem zarządzania profilem.

Na publicznym profilu:
  - pencil icons tylko przy content fields
  - About
  - Headline
  - Specialties
  - Certifications
  - Testimonials
  - Location/mode
  - Experience

Nie pokazuj inline edycji dla:
  - services/offers
  - pricing packages
  - visibility lifecycle controls
  - internal Coach Hub navigation

Services edytuje się w CoachServices / Coach Hub → Services.

═══════════════════════════════════════════════════════════════

NIE nazywać tego "Marketing", jeśli obejmuje personal profile.
