# P1 — Routing deep links, orphan routes, auth persistence

## Prerequisites

Prompt 14 recommended but not blocking. Canonical profile must still work.

---

## Current prod issues

### BUG — `Settings?tab=coach` ignored

```txt
URL: /Settings?tab=coach
Actual: Account / Trainee Profile form (Fitness Goal, Sports)
Expected: Coach Profile settings panel
```

Clicking sub-nav button **Coach Profile** works — only deep link / query param is broken.

### BUG — `/MyFollows` orphan route

```txt
/MyFollows → "Loading…" or nearly empty shell
```

Expected:

```txt
Redirect → /UserProfile?id=<canonical_self_id>&tab=follows
```

Menu row **Follows & followers** should use the same destination — not `/MyFollows`.

### BUG — `/Messages` and `/Teams` sometimes hit login wall

During QA sweeps, direct navigation to `/Messages` or `/Teams` redirected to login even when Home/Community were authenticated in the same session.

Likely causes:

- Route guard reads stale auth before hydration
- Storage state not shared on these routes
- `from_url` redirect loop after partial 429 failures

---

## Required fixes

### A — Settings tab sync

Support all of:

```txt
/Settings?tab=coach
/Settings?tab=account
/Settings?tab=billing
/Settings#coach
```

Rules:

- On mount: read `tab` query param OR hash → activate correct sub-panel
- On sub-nav click: update URL (`?tab=coach` or `#coach`) without full reload
- Browser back/forward must switch tabs correctly
- External links from Coach Hub / emails must land on Coach Profile panel

### B — MyFollows deprecation

Option 1 (preferred):

```js
// MyFollows page
redirect to `/UserProfile?id=${canonicalSelfId}&tab=follows`
```

Option 2: render embedded follows tab content at `/MyFollows` using same component as profile — but redirect is simpler.

Update menu:

```txt
Follows & followers → /UserProfile?id=<self>&tab=follows
```

Remove chevron if present. No submenu.

### C — Auth persistence on Messages / Teams

- `/Messages`, `/Teams`, `/Trainees` must use same auth provider as `/Home`
- Do not redirect to login while auth bootstrap is still loading — show app shell + spinner
- If session expired, redirect once with `from_url` — no loop
