## P0 — Szczegóły bugów

### UI-05 — Duplicate UserProfile / zły profil

**Symptom:** Po zapisie coacha widać inny profil (np. inne imię).  
**Root cause (bundle):** `filter(created_by, limit 50)` → wiele wierszy; `uu()` + `Fz()` scoring faworyzuje wiersz z najwyższym „coach score" (+1M pkt), niekoniecznie właściwy.  
**Brak w bundle:** `getCanonicalUserProfile` — nie wdrożone.  
**Prompt:** `prompts/base44-prompt-07-duplicate-profile-fix.txt`

### UI-03 — Coach Hub „You're not a coach yet"

**Symptom:** Po „Coach profile created" → pusty hub mimo COACH badge w menu.  
**Root cause:** Gate 1 (`coach_business_name`) OK → Gate 2 `getCoachStats()` fail → UI mapuje na `notCoach`.  
**Prod check (anon):** `POST /functions/getCoachStats` → **401** (endpoint istnieje; błąd 404 „Deployment does not exist" wymaga weryfikacji **zalogowanym** coachem).  
**Prompt:** `prompts/base44-prompt-05-coach-hub-getCoachStats.txt`

### UI-01 — Coach tab ostatni, default Overview

**Potwierdzone w bundle** — kolejność kluczy tabów w builderze:
```
overview → activity → follows → stats → coach
```
Default: `overview` (nie `coach` dla `is_public_coach`).  
**Oczekiwane:** `coach | overview | activity` (+ follows/stats dla self); default `coach`.  
**Prompt:** `prompts/base44-prompt-08-coach-ui-audit.txt` (Część A)

### UI-02 — Services: brak Usuń / Archive w UI

**Bundle:** `manageCoachServiceTier` — **TAK**  
**Bundle:** `visibility:"archived"` w UI — **NIE**  
**Bundle:** label „Archive offer" — **NIE**  
Karty ofert: Edit / View on profile / Copy link — bez archive flow.  
**Prompt:** `prompts/base44-prompt-08-coach-ui-audit.txt` (Część B)

### UI-04 — Menu: chevrony i Online flyout

**Znane z screenshotów użytkownika + handoff:**
- **Online** — chevron `>` ale panel statusu (Away/Busy/Offline) **niewidoczny** (overflow popover)
- **My Profile**, **Coach Hub** — chevron bez submenu (mylące)

**Reguła:** chevron **tylko** przy realnym flyout (Online).  
**Prompt:** `prompts/base44-prompt-04-menu-profile-hub.txt`
